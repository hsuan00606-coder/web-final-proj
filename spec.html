1. 系統簡介與架構

歐趴書局是一套二手教科書智慧拍賣網，採 Docker 全容器化部署。啟動後會同時運行四個容器：

容器名稱角色連接埠
web-backendFastAPI 後端＋前端網頁託管8000
mongodb資料庫（會員／商品／訂單）27017
chromadb向量資料庫8001
ollama-aiAI 大語言模型引擎（llama3.1，需 GPU）11434

專案資料夾位置：D:\used-textbook-system

2. 前置需求（僅第一次使用前需確認）

  • 已安裝 Docker Desktop（Windows 版）。
  • NVIDIA 顯示卡與驅動：AI 引擎 ollama-ai 需要 GPU 加速；若無 GPU，AI 對話會自動退回「測試模式」。
  • 專案檔案完整：確認 D:\used-textbook-system 內含 docker-compose.yml、deploy.bat、backend\、books_init.json、users_init.json。
  • 網路通暢：第一次會下載約 4.7GB 的 llama3.1 模型，需數分鐘。

3. 開機後啟動步驟

1

啟動 Docker Desktop

開機後先開啟「Docker Desktop」，等待左下角的鯨魚圖示變為綠色（狀態顯示 Running / Engine running）。容器必須由 Docker 引擎執行，這一步沒就緒後面都無法啟動。

2

開啟專案資料夾

用檔案總管進入 D:\used-textbook-system。

3

執行一鍵部署腳本

直接雙擊 deploy.bat 即可。它會自動完成下列三件事：

  • 建置並啟動全部四個容器（docker-compose up -d --build）
  • 等待 MongoDB 就緒後，匯入初始會員與商品資料
  • 初始化 / 載入 Ollama 的 llama3.1 模型

若想手動執行，可在此資料夾開啟終端機（PowerShell 或 CMD）依序輸入：

# 1) 建置並背景啟動所有容器
docker-compose up -d --build

# 2) 等待約 5 秒讓 MongoDB 就緒，再匯入初始資料
docker cp users_init.json mongodb:/tmp/users_init.json
docker cp books_init.json mongodb:/tmp/books_init.json
docker exec -i mongodb mongoimport --db obooks_db --collection users --drop --file /tmp/users_init.json --jsonArray
docker exec -i mongodb mongoimport --db obooks_db --collection books --drop --file /tmp/books_init.json --jsonArray

# 3) 載入 AI 模型（第一次會下載，較久）
docker exec -it ollama-ai ollama run llama3.1 "你好"

⚠️ mongoimport 帶有 --drop，會先清空再匯入。重複執行會把資料庫還原成初始狀態（清掉先前新增的商品／訂單）。日常只是重新開機啟動時，資料已存在則可略過匯入步驟，直接用步驟 4 的 up -d 即可。

4

日常啟動（資料已存在時）

若先前已部署過、只是電腦重開機，資料都還在，不需要重新匯入。在專案資料夾開終端機輸入：

docker-compose up -d

等所有容器狀態變為 Up / running 即可。可用以下指令確認：

docker ps

應看到 web-backend、mongodb、chromadb、ollama-ai 四個都在運行。

5

開啟瀏覽器使用網站

打開瀏覽器，輸入首頁網址：

http://localhost:8000/static/index.html

看到商品列表與左側 AI 助理即代表系統啟動成功。

5. 確認系統正常

  • 首頁能列出 5 本書且每本都有封面圖片。
  • 登入頁可用正確帳密登入；錯誤密碼會顯示「帳號或密碼錯誤」。
  • 首頁左側對 AI 助理輸入「我想找C語言」，右側會自動篩出 C 語言相關書籍。
  • 終端機檢查後端日誌：docker logs web-backend --tail 5 應顯示 Application startup complete。

6. 關閉系統

在專案資料夾的終端機輸入：

# 停止容器（保留資料）
docker-compose stop

# 或：停止並移除容器（資料仍保留於 volume）
docker-compose down

⚠️ 加上 -v（docker-compose down -v）會連同資料卷一併刪除，會清空所有會員、商品與訂單資料，非必要請勿使用。

7. （選用）分享給其他電腦使用

系統前端已使用相對路徑，可被其他裝置存取。臨時對外分享（免安裝）可用 SSH 反向通道：

ssh -o StrictHostKeyChecking=no -R 80:localhost:8000 nokey@localhost.run

執行後畫面會印出一組 https://xxxx.lhr.life 的公開網址，其他電腦／手機開此網址即可使用。

此通道隨終端機關閉而失效，重開會得到不同網址；本機電腦與容器須持續運行。正式長期使用建議改用具帳號的服務（ngrok、cloudflared）或部署到雲端主機。

8. 疑難排解

症狀處理方式
網頁打不開 / 連線被拒確認 Docker Desktop 已啟動，且 docker ps 看得到 web-backend。
商品列表空白重新匯入資料：執行步驟 3 的兩條 mongoimport 指令。
AI 回覆出現「【測試模式】」代表沒連到 Ollama。確認 ollama-ai 容器運行、GPU 可用，並已 ollama run llama3.1 載入模型。
埠號被占用（8000/27017…）關閉占用該埠的程式，或修改 docker-compose.yml 的對外埠號後重啟。
改了程式碼沒生效前端（HTML/JS/CSS）重整網頁即可；後端（main.py）需 docker restart web-backend。
