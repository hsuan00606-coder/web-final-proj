1. 系統架構總覽

系統由四個 Docker 容器組成，前端網頁由後端容器一併託管（前後端同源）：

容器角色對外埠
web-backendFastAPI 後端 + 前端靜態頁面託管8000
mongodb資料庫：會員 / 商品 / 訂單27017
chromadb向量資料庫（預留）8001
ollama-aiAI 大語言模型 llama3.1（需 GPU）11434
[ 瀏覽器 HTML/JS ] → common.js (apiFetch) → [ web-backend FastAPI main.py ] ├─→ [ mongodb ] 使用者 / 商品 / 訂單資料 └─→ [ ollama-ai ] AI 對話生成

2. 開機後啟動步驟

1

啟動 Docker Desktop

等待左下角鯨魚圖示變綠（Engine running）。

2

用 PowerShell 啟動本機端

在專案資料夾以 PowerShell 執行（取代有編碼問題的 deploy.bat）：

powershell -ExecutionPolicy Bypass -File D:\used-textbook-system\deploy.ps1

它會：建置並啟動容器 → 匯入初始資料（會員、5 本書）→ 暖機 llama3.1。

若先前已部署過、資料還在，只是重開機：直接 docker-compose up -d 即可，不必重新匯入。

3

開啟網站

http://localhost:8000/static/index.html
4

（選用）取得外網網址

powershell -ExecutionPolicy Bypass -File D:\used-textbook-system\geturl.ps1

會輸出一行 https://xxxx.lhr.life/static/index.html 供其他電腦訪問。

3. 每個檔案的功能說明

後端 backend/

檔案功能
main.pyFastAPI 後端核心，定義所有 API（會員認證、商品 CRUD 與搜尋、訂單、AI 對話），並託管前端靜態頁與商品圖片。連線 MongoDB 與 Ollama。
requirements.txtPython 套件依賴清單（fastapi、uvicorn、pymongo、bcrypt、python-multipart、pydantic、requests）。
Dockerfile後端映像檔建置流程（安裝套件、複製程式、以 uvicorn 啟動）。
static/uploads/商品圖片存放處（01~05.jpg 為初始書封，賣家上傳的圖也存在此）。

前端共用 backend/static/

檔案功能
common.js所有頁面共用的 JS：API 基底路徑、apiFetch 請求工具、Auth 登入狀態、Cart 購物車（localStorage）、I18n 三語系、共用導覽列產生器。
common.css全站共用樣式（橘色主題、導覽列、按鈕、卡片等）。

前端頁面 backend/static/

頁面功能
index.html首頁：商品列表 + 左側 AI 助理對話。可關鍵字搜尋；對 AI 輸入「我想找C語言」會連動篩選右側商品。
product.html商品詳情頁：大圖、價格、數量、規格/目錄/評論分頁、加入購物車、立即購買；最下方「你可能也喜歡」每 4 秒輪播。
seller.html賣家專區：新增 / 編輯商品（書名、作者、定價、敘述、標籤、上傳圖片含預覽）。需登入。
cart.html購物車：列出商品、調整數量、移除、計算小計與運費，前往結帳。需登入。
checkout.html結帳：填配送與付款資訊，計算總計（小計+運費+稅務）並送出訂單。需登入。
order_result.html訂單結果 / 查詢：顯示訂單明細、金額、物流進度條；可輸入訂單編號查詢。
orders.html歷史訂單：列出所有訂單與狀態，可查看詳情。需登入。
log_in.html會員登入 / 註冊（實際使用，對應 /api/v1/auth/*）。使用中
about.html關於作者：作者資訊與專案心得（由導覽列 🧑‍🎓 icon 進入）。
login.html舊版登入頁，呼叫不存在的端點，未使用。未使用
upload.html舊版上架頁，未使用（已由 seller.html 取代）。未使用

檔案功能
docker-compose.yml四個容器的編排設定（映像、埠、網路、資料卷、GPU）。

4. API 路由一覽

方法路徑功能資料來源
POST/api/v1/auth/register註冊會員（bcrypt 雜湊密碼）users
POST/api/v1/auth/login登入驗證，回傳 tokenusers
GET/api/v1/products商品列表 / 關鍵字搜尋（可帶 ?q=）books
GET/api/v1/products/{id}單一商品詳情books
POST/api/v1/products新增商品（multipart 含圖片）books
PUT/api/v1/products/{id}編輯商品（multipart）books
POST/api/v1/orders建立訂單orders
GET/api/v1/orders所有訂單orders
GET/api/v1/orders/{id}單一訂單查詢orders
POST/api/v1/chatAI 對話ollama-ai

5. 串接的資料流

① 會員登入 / 註冊

log_in.html（輸入帳密） → common.js apiFetch → POST /api/v1/auth/login → main.py 比對 users 集合（bcrypt 驗證密碼） → 回傳 token → 前端存入 localStorage（Auth.save）

② 商品瀏覽與搜尋

index.html loadProducts() → GET /api/v1/products?q=關鍵字 → main.py 萃取關鍵字（去除「我想找/推薦…」贅詞） → 比對 books 的書名/作者/敘述/標籤 → 回傳商品陣列 → 前端渲染商品卡片

③ AI 助理對話（連動商品）

index.html handleAiChat() → POST /api/v1/chat { session_id, message } → main.py 帶 system prompt → ollama-ai llama3.1 生成回覆 → 回傳 reply → 顯示於對話框 → 同時以使用者訊息呼叫 loadProducts(msg) 連動篩選右側商品

若 Ollama 連線失敗或逾時，後端自動回「測試模式」訊息，確保前端不中斷。

④ 賣家新增 / 編輯商品

seller.html（表單 + 圖片）→ FormData（multipart） → POST /api/v1/products 或 PUT /api/v1/products/{id} → main.py 儲存圖片到 static/uploads/，寫入 / 更新 books 集合 → 回傳商品 → 列表即時刷新（買家端搜尋即可找到）

⑤ 購物車 → 結帳 → 訂單

加入購物車（index/product）→ Cart 存於 localStorage（純前端） → cart.html 檢視 / 調整數量 → checkout.html 計算 總計 = 小計 + 運費 + 稅務 → POST /api/v1/orders（帶 items/配送/付款/金額） → main.py 產生訂單編號、寫入 orders 集合 → 清空購物車 → 導向 order_result.html → GET /api/v1/orders/{id} 顯示訂單明細與物流進度

7. 疑難排解

症狀處理
網頁打不開確認 Docker Desktop 已啟動，docker ps 看得到 web-backend。
商品列表空白重新匯入：重跑 deploy.ps1 的匯入步驟。
AI 回覆出現「【測試模式】」未連到 Ollama；確認 ollama-ai 運行、GPU 可用、已載入 llama3.1。
改了程式沒生效前端（HTML/JS/CSS）按 Ctrl+F5 強制重整；後端（main.py）需 docker restart web-backend。
導覽列新 icon 沒出現瀏覽器快取了舊 common.js，按 Ctrl+F5。
