「歐趴書局」二手教科書拍賣網 系統規格計畫書

部署模式：Docker 全容器化 (Full Containerization) | 資料庫：純 MongoDB 架構

一、 專案概述與核心目標

本專案旨在開發一個名為「歐趴書局」的二手教科書智慧拍賣與媒合網站。系統整合現代全端架構與 AI Agent 技術，提供使用者直覺的書籍搜尋、多輪對話智慧推薦、以及完整的二手書上架交易功能。本專案最終採取全容器化（Full Containerization）架構，將所有後端、資料庫與 AI 推理引擎納入 Docker Compose 編排，透過一個自動化腳本即可在具備 NVIDIA GPU 環境的本機端快速完成一鍵部屬，提供流暢的期末實機 Demo 體驗。

二、 系統技術棧與版本規範 (Tech Stack)

元件層級 選用技術/套件 指定版本 容器化角色 / 說明

四、 全容器化架構設計細節

為了達成「一鍵自動化部署」，本系統由四個 Docker 容器於同一個虛擬網域中協同運作：

  1. web-backend 容器： 執行 FastAPI 服務，對外開放 8000 連接埠。瀏覽器訪問 http://localhost:8000/static/index.html 即可讀取網頁。
  2. mongodb 容器： 對內開放 27017 連接埠。掛載 Docker Volume 進行資料持久化。用戶資料（包含經 bcrypt 雜湊之密碼）與商品元數據均儲存於此，利用原生的 ObjectId 進行一對多（1:N）賣家與書籍之關聯。
  3. chromadb 容器： 對內開放 8000 連接埠。FastAPI 透過 chromadb.HttpClient(host="chromadb") 連線，避免因多行程鎖定檔案或容器重啟導致資料遺失。
  4. ollama-ai 容器： 對內開放 11434 連接埠。透過 Docker Compose 的 deploy.resources.reservations.devices 機制，強制預留並掛載宿主機的 NVIDIA GPU 以啟動 CUDA 加速。

五、 關鍵功能與核心規格細節

1. 前端靜態路由託管 (方案 A)

專案目錄結構採取內嵌式靜態資源管理。前端所有頁面（包含 index.html、樣式、腳本）均放置於後端專案目錄下的 static/ 資料夾中。FastAPI 啟動時透過以下邏輯掛載：

app.mount("/static", StaticFiles(directory="static"), name="static")

此設計可確保前後端服務處於同一個連接埠（Port 8000），徹底消除跨來源資源共用（CORS）的複雜設定，最適合本機快速 Demo。

2. 安全認證與預設測試帳密

使用者註冊與登入均採用 bcrypt 進行密碼雜湊加密。為方便評審與老師免去註冊手續直接進行功能展示，系統初始化時會自動在 MongoDB 寫入以下兩組已雜湊的測試帳號：

  • 測試買家帳號： buyer@test.edu.tw / 密碼：password123
  • 測試賣家帳號： seller@test.edu.tw / 密碼：password123

3. 極致流暢的註冊跳轉體驗

為避免不必要的等待，註冊功能的防呆優化升級為「毫秒級即時跳轉」。後端 API 完成資料校驗與寫入 MongoDB 後，立即回傳成功響應，前端 JavaScript 收到 Response 後不做任何非必要延遲，立刻引導使用者跳轉至登入頁面。

4. 輕量化多輪對話 Session 機制

智慧 AI 助理支援多輪對話與商品回溯（記憶前一輪推薦的商品 ID 列表）。為了降低架構開銷，此 Session 暫存於 FastAPI 容器的記憶體（In-Memory Python Dict）中。只要使用者關閉網頁瀏覽器或容器重啟，多輪對話與回溯功能即自動失效清空，此符合本機 Demo 之實作預期。

六、 自動化部屬與環境初始化指南 (Automation Script)

本專案提供 Windows 環境專用的自動化部署指令檔 deploy.bat。評審機台只需預先安裝好 Docker Desktop、NVIDIA 驅動程式 以及 NVIDIA Container Toolkit，即可一鍵完成部屬：

1. 一鍵啟動指令

雙擊執行專案根目錄下的 deploy.bat 或在終端機輸入：

.\deploy.bat

2. 自動化腳本執行流程

指令檔在執行時會依序自動完成以下任務：

  1. 執行 docker-compose up -d --build，在背景建立並啟動四個核心服務容器。
  2. 監聽 mongodb 容器狀態，當資料庫就緒後，自動執行 mongoimport 指令，將專案內的 books.json 初始二手書商品資料與預設測試會員資料倒進資料庫。
  3. 監聽 ollama-ai 容器狀態，自動在容器內觸發 ollama run llama3.1，從官方線上下載 Llama 3.1 (8B) 模型至容器掛載的 Volume 中，並完成初始化。

3. 驗證部屬成果

當腳本顯示 [SUCCESS] 歐趴書局全容器化環境部屬完成！ 後，即可打開瀏覽器訪問以下網址進行 Demo：

http://localhost:8000/static/index.html

七、 系統清理與重置 (Reset & Cleanup)

若需要徹底清除快取、重置資料庫或卸載容器，請執行下列指令：

docker-compose down -v

加上 -v 參數將會一併刪除 MongoDB 與 ChromaDB 的數據資料夾（Volume），確保下一次執行部署腳本時，系統能以 100% 全乾淨的狀態重新載入 books.json。
